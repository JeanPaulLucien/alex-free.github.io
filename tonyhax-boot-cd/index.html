Wait What? I Don't Need A Supported Game Or Even A Memory Card? What's The Catch?

The catch is the easist consoles to use the TonyHax Boot CD on are the super ultra rare early American ones with a serial number lower then 592xxx (according to the OG Swap Trick Guide from 1996). My Playstation was manufactured in October of 1995, serial number 5768605 and this works perfectly.

I Have An Early PSX Console With A Serial Number Lower Then 592xxx, What Do I Do?

Okay, okay. So this is the ingenious idea around this boot cd. These early consoles are susceptible to the METHOD ONE: The Audio Menu [Safe and very easy] swap trick. For the past 24 years this had one issue, the original TOC of the legit game disc would be used. This mostly worked but you'd get issues with some games not booting or messed up audio, and you'd have to hot swap trick again if it was a multi-disc game at some point (many people incorrectly said playing multi-disc PSX games with just the swap method was not possible!). But by using TonyHAX, you can swap the TonyHax boot cd using this method with any legit game and then once the TonyHax Boot CD is booted you can swap your backup/homebrew! Finally, this is how you pull it off:

  • Turn on the Playstation 1 with No CD/game inserted and select the CD Player in the main menu.
  • Open the CD drive. Place any legit game disc that the console can read normally into the drive.
  • Now you need to block the lid sensor which is a circle to the right of the CD drive so that the Playstation thinks the lid is closed. In my opinion, the easist thing to do is to take a hunk of aluminum foil and mold it to the right size to the length from the pointy bit which normally would press down this sensor to the sensor button in the fully depressed state. You just need the PSX to think the lid is closed. Once you do this, the legit game disc will spin for a second, and then suddenly stop and display at least one track in the CD Player menu.
  • Okay, so right now your Playstation should have a legit game disc which is not moving and stopped in the CD drive, and something blocking the sensor so the console thinks the CD drive is still closed. Now, remove that legit game disc and replace it with your burned TonyHax Boot CD. Use the controller to then exit the CD Player menu, and wait. The PSX will show the black license screen and proceed to boot the TonyHAX Boot CD.
  • Now you can remove whatever you jammed into that lid sensor, wait for the burned TonyHax Boot CD to stop spinning, remove it, and put in the burned backup game or homebrew CD you actually want to play! Then close the Playstation's lid, and play like normal.

What's The Best Way To Burn This TonyHax Boot CD, And My Backups/Homebrew?

I can not recommed cdrdao enough. This is an open source command line program that makes it easy to burn BIN+CUE files. With any program however, you have to burn as slow as possible. Some drives can only go soo slow, I luckily have some PowerPC Macs which do as low as 8x speed. I haven't ran into issues however on my semi-modern HP laptop running Fedora linux which does as low as 10x. If you burn at full speed (like 24x) the PSX will not read your CD-Rs! Also, use high quality CD-Rs like Verbatim! Don't go cheap on a console this old, you'll probably regret it and waste $$$.

You should use these arguments with cdrdao:

cdrdao write --swap --speed 1 --driver generic-mmc tonyhax-boot-cd-1.3.5.cue

The --swap argument is not really neccesary for burning the TonyHax boot cd itself, but it is for PSX games that contain CD audio and is good practice to use by default.

Note: You'll probably have to add a --device argument if this is not automatically detected. For PowerPC Macs this is either IODVDServices or IOCompactDiscServices. On Fedora Linux it's /dev/cdrom.

Of course other programs will probably work, but make sure you burn at the slowest speed your drive supports!.

My Playstation Won't Boot Legit Games Or Will Boot Legit Games But Not My Burned CDs?

At least for my specific console, I need to move my PSX vertically or at the angle of this slash: \ for it to boot any burned CD-R games/homebrew. I don't have to do this for burned audio CDs, don't know why. Please try this if your Playstation is not booting to the black license screen or isn't booting your game after swapping it on the TonyHax screen.

What If I Want To Build The CD BIN+CUE Myself?

You need Lameguy64's MKPSXISO. Next you need to place the file LICENSEA.DAT in the same directory as the tonyhax.exe to license the built images. The LICENSEA.DAT file can be obtained from the official PSX PSYQ SDK.

Then, just cd into wherever the tonyhax-boot-cd directory is. Next execute mkpsxiso us-cd.xml.
